Implement the TipTap rich-text editor component with collaborative editing for Ship documents. This is the core editing experience used across all document types (wiki, issue, project, program, weekly plans, etc.).

Create `src/components/Editor.tsx` (or `UnifiedEditor.tsx`) that initializes a TipTap editor with StarterKit, plus these extensions: Collaboration (Yjs binding to a shared Y.Doc fragment named 'default'), CollaborationCursor (awareness with user name/color), Placeholder, Link, TaskList+TaskItem, Table+TableRow+TableCell+TableHeader, CodeBlockLowlight, Dropcursor. Connect to the WebSocket collaboration server at `ws://localhost:3000/collaboration/{docType}:{docId}` using y-websocket's WebsocketProvider. Use y-indexeddb for offline persistence (IndexedDB provider). Handle the cache-clear message (type 3) from the server by clearing IndexedDB before sync.

Create custom extensions: MentionExtension (trigger with @, fetch suggestions from /api/search/mentions, render as inline chip linking to the mentioned document/person), ImageUpload (drag-and-drop/paste images, upload to /api/files, insert as resizable image node), FileAttachment (upload non-image files, render as download link), SlashCommands (trigger with /, show a menu of block types: heading, bullet list, numbered list, task list, code block, table, image, divider), HypothesisBlock (custom node for embedding plan/hypothesis content that gets extracted server-side).

Create `src/components/editor/BacklinksPanel.tsx` that fetches backlinks for the current document from /api/documents/:id/backlinks and displays them as a list below the editor. Create `src/components/editor/CommentDisplay.tsx` for inline comments (highlight marked text, show comment thread in a popover, support replies and resolve).

Create a properties sidebar that renders next to the editor via React Portal into `#properties-portal`. The sidebar shows document metadata: type, status (for issues), priority, assignee (PersonCombobox), belongs_to associations (program/project/sprint chips), visibility toggle, created/updated timestamps. Changes in the sidebar call PATCH on the document API and update via TanStack Query mutation.

Wire the editor into the document page: when navigating to /documents/:id, fetch the document, initialize the Yjs provider, render the editor with the properties sidebar. Handle WebSocket close codes: 4100 (document converted, redirect to new doc), 4101 (content updated via API, reconnect), 4403 (access revoked, show error).

Verify: open a document, type text, confirm it persists (reload and content appears), open the same document in two tabs, type in one, confirm text appears in the other within 1 second, confirm cursor presence shows the other user's name and color.
