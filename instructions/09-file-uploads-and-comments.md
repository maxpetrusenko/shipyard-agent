Implement file upload and inline comments systems for Ship.

For file uploads, create `routes/files.ts` with auth middleware: POST /api/files/upload-request (validates filename/mimeType/sizeBytes with Zod, blocks dangerous extensions like .exe/.bat/.sh/.dll/.jar/.ps1, creates a files row with status=pending, returns a presigned S3 upload URL if S3_UPLOADS_BUCKET is configured or a local upload endpoint for dev), POST /api/files/upload/:id (for local dev: accepts multipart form data, writes to uploads/ directory, updates files row to status=uploaded), GET /api/files/:id (returns file metadata + download URL). For S3: use @aws-sdk/client-s3 and @aws-sdk/s3-request-presigner with 15-minute URL expiration. For local dev: serve uploads/ as static files. Set max file size to 1GB. Validate UUID format on all ID params to prevent path traversal.

For comments, create `routes/comments.ts` with: GET /api/documents/:id/comments (list all comment threads for a document, grouped by comment_id), POST /api/documents/:id/comments (create a new comment thread with comment_id matching the TipTap comment mark, or reply to existing via parent_id), PATCH /api/comments/:id (update comment content), POST /api/comments/:id/resolve (set resolved_at timestamp), DELETE /api/comments/:id (only author or admin can delete). Create `routes/backlinks.ts` with: GET /api/documents/:id/backlinks (query document_links where target_id matches, return source documents with title and type), POST /api/documents/:id/links (upsert document_links from extracted mentions/links in content).

On the frontend, create `src/services/upload.ts` with: requestUpload (calls upload-request endpoint), uploadFile (either uploads directly to presigned S3 URL or to the local endpoint), trackUploadProgress. Create `src/services/uploadTracker.ts` to track active uploads and warn on navigation (UploadNavigationWarning component). Create `src/hooks/useCommentsQuery.ts` for fetching and mutating comments via TanStack Query.
