Set up the PostgreSQL database layer for Ship. Create a `db/` directory under the API package with: `client.ts` (pg Pool with connection pooling, idle timeout, statement timeout, graceful shutdown on SIGTERM/SIGINT), `schema.sql` (the full schema), and a `migrations/` folder for incremental changes.

The schema must implement Ship's "everything is a document" model. Core tables: `workspaces`, `users` (with email, password_hash, x509_subject_dn, is_super_admin), `workspace_memberships` (role: admin|member), `sessions` (TEXT primary key from crypto.randomBytes, with last_activity and ip_address), `documents` (unified table with document_type ENUM for wiki/issue/program/project/sprint/person/weekly_plan/weekly_retro/standup/weekly_review, JSONB properties, JSONB content for TipTap, BYTEA yjs_state, parent_id self-reference with circular-reference prevention trigger, visibility private|workspace, soft-delete via deleted_at, ticket_number auto-increment per workspace), `document_associations` (junction table with relationship_type ENUM: parent/project/sprint/program), `document_history`, `document_snapshots`, `audit_logs`, `comments` (threaded with parent_id), `files`, `document_links` (for backlinks), `api_tokens`, `workspace_invites`, `oauth_state`, `sprint_iterations`, and `issue_iterations`. Add pg_trgm extension and comprehensive indexes (GIN on properties, trigram on titles, partial indexes on archived_at/deleted_at).

Write a `seed.ts` that creates a demo workspace, admin user (password hashed with bcryptjs), and sample documents. Write a `migrate.ts` that applies SQL files from `migrations/` in order. Verify by running `pnpm db:migrate` and `pnpm db:seed` against a local PostgreSQL instance, then querying each table to confirm it exists with correct columns.
