# Runs Page Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a dedicated `Runs` page in the top nav that shows refactoring-oriented run history using the existing run data.

**Architecture:** Add a new server-rendered `/runs` page alongside the existing dashboard, settings, and benchmarks pages. Reuse `InstructionLoop.getRunsForListingAsync()` and the shared top-nav helpers, then filter the existing run records toward code/refactoring work by default so pure ask-only chats do not dominate the page.

**Tech Stack:** TypeScript, Express, existing server-rendered HTML pages, Vitest
